# React 모범 사례 가이드

React 애플리케이션을 개발할 때 따라야 할 주요 모범 사례들을 알아보겠습니다.

## 1. 컴포넌트 설계 원칙

### 단일 책임 원칙
각 컴포넌트는 하나의 명확한 역할만 가져야 합니다.

```tsx
// ❌ 나쁜 예
function UserProfileCard({ user, onEdit, onDelete, showStats }) {
  return (
    <div>
      <img src={user.avatar} alt={user.name} />
      <h3>{user.name}</h3>
      <p>{user.email}</p>
      {showStats && <UserStats userId={user.id} />}
      <button onClick={onEdit}>편집</button>
      <button onClick={onDelete}>삭제</button>
    </div>
  );
}

// ✅ 좋은 예
function UserAvatar({ src, alt }) {
  return <img src={src} alt={alt} />;
}

function UserInfo({ name, email }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>{email}</p>
    </div>
  );
}

function UserActions({ onEdit, onDelete }) {
  return (
    <div>
      <button onClick={onEdit}>편집</button>
      <button onClick={onDelete}>삭제</button>
    </div>
  );
}
```

## 2. State 관리

### useState vs useReducer
복잡한 상태 로직에는 `useReducer`를 사용하세요.

```tsx
// 복잡한 상태 관리
function TodoApp() {
  const [state, dispatch] = useReducer(todoReducer, initialState);
  
  return (
    <div>
      <TodoList todos={state.todos} dispatch={dispatch} />
      <AddTodo dispatch={dispatch} />
    </div>
  );
}
```

## 3. 성능 최적화

- **React.memo**: 불필요한 리렌더링 방지
- **useMemo**: 비용이 큰 계산 결과 메모이제이션
- **useCallback**: 함수 참조 안정화

> **팁**: 성능 최적화는 실제 성능 문제가 발생했을 때 적용하세요. 조기 최적화는 피하는 것이 좋습니다.
